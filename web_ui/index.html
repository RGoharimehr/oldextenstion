How it works

After each simulation step the Omniverse extension reads these settings and colours USD prims in the 3D viewport using the selected property and colormap.

The colorization uses strongerThanDescendants binding on the parent Xform, so all child geometry is covered automatically.

When no data is available for a prim the fallback material /World/Looks/Aluminum_Foil is applied (or silver if not present in the stage).

1 · Add Flownex Attribute to Prims

Adds a flownex:componentName attribute to every prim under the target path so you can assign component identifiers in the Property panel.

2 · Generate Mapping Config File

Matches prims whose flownex:componentName value equals a ComponentIdentifier in Outputs.csv and writes FlownexMapping.json to the IO directory.
